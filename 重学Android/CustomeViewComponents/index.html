自定义 View

创建自定义View 的步骤： 1. 继承 View 创建你自己的类 2. 提供构造函数，自定义属性 3. 创建事件的监听 4. override onMeasure 和 onDraw 方法。默认的 onDraw 什么都不干， onMeasure 默认设置 100 X 100 的大小 5. 按需 override 其他的 on... 方法

Override onDarw 和 onMeasure

onDraw 方法传入一个 Canvas，你可以在 Canvas 上实现你的功能：2D作图、文字式样等任何你能想到的东西。 > Canvas 不适用于 3D 作图。你可以继承 SurfaceView ，并且在独立的现成中进行绘图，来实现3D的图形。

onMeasure 是 View 本身和它的 Container 之间进行 View 尺寸约定的重要方法。onMeasure 需要高效又精确的报告它所包含的部分的测量结果。由于它的 Container 的限制、并且 setMeasuredDimension 需要传入计算过的宽度和高度，这让 onMeasure 的实现有些复杂。在 onMeasure 中必须调用 setMeasuredDimension 来对外通知它的计算结果。 从更高的角度来看， onMeasure 的调用过程概括如下： 1. 调用 onMeasure 方法，并传入宽和高的限制。 2. 在 onMeasure 中计算测量过的宽和高，这是渲染当前组件所必须的参数。它的宽和高应该保持在传入的限制之内。当然你也可以大于传入的限制（这时它的 Container 会选择对于这种情况应该如何处理，例如剪切、滚动、抛出异常、传入不同的参数重新调用 onMeasure 方法）。 3. 宽和高计算完成后，必须调用 setMeasuredDimension 方法传入计算好的值。

其他常用的方法： | 分类 | 方法 | 说明 | | ---------------- | ----------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- | | Creation | 构造方法 | 两种形式：1 从代码调用构造方法 2 通过layout 文件 inflate View，这里需要解析和应用你再布局文件中定义的属性 | | | onFinishInflate | 在View自身和它的子View 全部 inflate 完成之后调用 | | Layout | onMeasure | 决定 view 的尺寸需求 | | | onLayout | 在需要给子View 分配尺寸和位置的时候调用 | | | onSizeChanged | 当自己的尺寸发生变化的时候调用 | | Drawing | onDraw | 在View 需要渲染其中的内容时调用 | | Event processing | onKeyDown | 当发生新的按键事件时调用 | | | onKayUp | 按键抬起时调用 | | | onTrackballEvent | Trackball 事件发生时调用 | | | onTouchEvent | 触摸事件发生时调用 | | Focus | onFocusChanged | 当得到或失去焦点时调用 | | | onWindowFocusChanged | 当所在窗口获得或失去焦点时调用 | | Attaching | onAttachedToWindow | 当View attach 到 Winddow 的时候调用 | | | onDetachedFromWindow | 当 View 从 Window detach 的时候调用 | | | onWindowVisibiityChanged | 当包含View的窗口的可见性改变的时候调用 |

Compound Controls

如果你不希望从头创建一个新View，而是想重用已有的组件，那么就创建一个复合组件。简单的说就是把更多的原子操作的能力增加给逻辑上的一组组件，这样他就可以被视为一个组件。比如一个下拉框可以认为是由一个EditText、按钮和 PopupList 组成的。 创建复合组件的步骤： 1. 创建ViewGroup的子类。 2. 在构造函数中接受父类需要的参数，通过父类的构造函数传递给父类。你也可以通过XML定义你自己的属性。 3. 你也可以创建自己的事件的监听器，自定义对事件的响应。 4. 你还可以创建你自己的属性，比如给EditText增加初始值。 5. 如果你继承了一个 Layout ，你可能不需要再 override onDraw 和 onMeasure 方法，因为 Layout 已经默认实现了这两个方法。 6. 你可能还需要 override 其它的 on... 方法来完成你的功能。

继承 Layout 有如下优点： - 可以使用 XML 指定布局，也可以在代码中创建视图 - 默认的 onDraw 和 onMeasure 方法可能已经满足需求，无需 override 他们 - 把它作为普通的组件重用

Creating a custom view class

定制 View 的原则： - 遵守 Android 标准 - 提供 Android XML 布局使用的定制 styleable 属性 - 发送辅助事件（accessibility） - 兼容多个 Android 系统平台

Define Custom Attributes

自定义 View XML 属性的定义方式： - 使用 <declare-styleable> 定义属性

- 在 XML 布局中指定属性的值 - 运行时获取属性值，当从XML创建View的时候，所有的XML属性通过 AttributeSet 参数传递给构造方法。这样就可以从 AttributeSet 中获取属性值，但那是有两个缺点 - 属性值中引用的其他资源并没有处理 - 式样没有应用 你可以这样处理：将 AttributeSet 传给 obtainStyledAttributes， 这个方法帮你返回已处理好引用资源和式样的 TypedArray。 - 使用获取的属性值。属性的值在View 初始化的时候设置。你可以给定制属性增加对应的方法。

Design For Accessibility

如果你的定制 View 需要支持各种用户，你应该增加对残疾人的支持： - 使用 android:contentDescription 来标记输入框 - 使用 sendAccessibility 发送辅助时间 - 支持 D-pad 和 trackball
